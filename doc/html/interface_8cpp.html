Interface function for calling GAIM from within any other C/C++ source code of software (libgaim).

Interface functions (to use GAIM through other programming languages.

It initializes all the necessary parameters and vectors for evolving a population of individuals based on a predetermined Genetic Algorithm (GA).

Parameters
[in]funcA pointer function to the objective/fitness function
[in]n_generationsNumber of generations
[in]population_sizeNumber of individuals within a population
[in]genome_sizeHow many parameters the fitness function has
[in]n_offspringsNumber of new candidate individuals (children)
[in]n_replacementsNumber of parents to be replaced per generation
[in]n_roundsNumber of independent experiments to run
[in]aVector with genes lower bounds
[in]bVector with genes upper bounds
[in]clippingGene's clipping method (universal: all the genes are clamped with the same values a and b. Individual: Each gene has its own clamped values a and b.
[in]clipping_fnameFilename of clipping file
[in]selection_methodSelection method must be one of the following: ktournament, truncation, linear_rank, random, roulette, stochastic_roulette, whitley
[in]biasThis is the bias term when the Whitley selection method is used
[in]num_parentsDetermines how many parents will be selected from the population
[in]lower_boundDetermines the starting index when the Truncation selection method is used
[in]kDetermines the tournament size (number of individuals to choose) when the k-tournament method is used
[in]replaceIf it's true then any selection method will choose the parents from the original population with replacement. param[in] crossover_method Crossover method must be one of the following: one_point, two_point, uniform, flat, discrete, first_order
[in]mutation_methodMutation method muse be one of the following: delta, random, nonuniform, fusion, swap_mutation
[in]mutation_rateThe mutation rate when the delta mutation method is used
[in]mutation_varThe mutation variance when the delta mutation method is used
[in]low_boundDetermines the lower bound for the uniform distributions used in all mutation methods
[in]up_boundDetermines the upper bound for the uniform distributions used in all mutation methods
[in]orderDefines the order of the Nonuniform delta mutation
[in]is_realDetermines if the mutation operators will operate on real or integer numbers
[in]is_im_enabledEnables/disables the island model operation
[in]n_islandsNumber of islands in a Island Model
[in]n_immigrantsNumber of individuals to move from one island to another (only for island model)
[in]migration_intervalDetermines how frequent individuals have to move from one island to another
[in]pickup_methodSelection method of migrating individuals in an island model, poor: Select individuals with lowest fitness, elite: select individuals with the highest fitness, and random: select individuals randomly
[in]replace_methodReplacement method of individuals on islands who receive immigrants. Three method (poor, elite, and random).
[in]im_graph_fnameThe filename of the file that contains the connectivity graph for an island model.
[in]experiment_idA name for any optimization (experiment). GAIM will store all the parameters of the optimization in a file with that name
[in]log_pathThe directory in which all the results and logs will be saved
[in]return_typeA string that determines which run the genome will be returned. The available options are: (*) "minimum" - the genome with the minimum Euclidean distance is returned (*) "maximum" - the genome with the maximum Euclidean distance is returned (*) "random" - a randomly chosen genome is returned
[in]log_fitnessEnable/disable the track of individuals' fitness
[in]log_average_fitnessEnable/disable the track of population (average) fitness
[in]log_bsfEnable/disable the track of Best So far Fitness (BSF) fitness.
[in]log_best_genomeEnable/disable the track of the best genome
Returns
res A ga_results_s data structure that contains the average fitness, the BSF, and the best genome found from the GA.
